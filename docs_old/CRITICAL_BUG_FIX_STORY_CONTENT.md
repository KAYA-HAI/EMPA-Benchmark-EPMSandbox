# 🔴 严重Bug修复：IEDR Prompt 缺失完整故事内容

**日期**: 2025-10-30  
**严重等级**: 🔴 Critical  
**影响**: 所有IEDR评估（已完成的100个和未来的评估）

---

## 问题发现

用户发现了一个**严重的数据缺失问题**：IEDR评估的 prompt 中**只使用了"故事的结果"字段（~95字符），完全遗漏了"故事的经过"的所有4个阶段（~1000字符）！**

---

## 问题分析

### 数据结构（character_stories.json）

每个剧本的完整结构：

```json
{
  "剧本编号": "script_001",
  "故事的经过": {
    "阶段1": {
      "标题": "引发回忆",
      "内容": "... (~266字符)"
    },
    "阶段2": {
      "标题": "引发思考",
      "内容": "... (~247字符)"
    },
    "阶段3": {
      "标题": "引发自审",
      "内容": "... (~253字符)"
    },
    "阶段4": {
      "标题": "情绪崩发",
      "内容": "... (~202字符)"
    }
  },
  "故事的结果": "... (~95字符)",
  "故事的插曲": "... (~101字符)"
}
```

**总计**: 约 1164 字符的关键剧情信息

### 修复前的 Prompt（错误版本）

```python
## Scenario (场景信息)
剧本编号: {scenario.get('剧本编号', 'N/A')}
故事结果: {scenario.get('故事的结果', 'N/A')}  # ❌ 只有95字符！
```

**缺失内容**:
- ❌ 阶段1: 引发回忆 (~266字符)
- ❌ 阶段2: 引发思考 (~247字符)
- ❌ 阶段3: 引发自审 (~253字符)
- ❌ 阶段4: 情绪崩发 (~202字符)
- ❌ 故事的插曲 (~101字符)

**缺失比例**: ~92% 的故事内容被遗漏！

---

## 影响分析

### 对 IEDR 评估的影响

这些被遗漏的信息对各个维度的评估至关重要：

#### 1. **A.1 情绪强度** - 🔴 严重影响
- **阶段4: 情绪崩发**是判断情绪强度的**关键证据**
- 例如: "积压已久的情绪彻底爆发了。愤怒、委屈、不甘，像潮水一样将林晚淹没"
- **没有这个信息**，无法准确判断是 [2] 强烈 还是 [3] 极端

#### 2. **C.1 处境复杂性** - 🔴 严重影响
- **阶段1-3**展示了角色的完整经历和心理过程
- 需要通过这些阶段判断是否涉及"角色独特的、复杂的个人经历" ([3]级)

#### 3. **P.1 初始能动性** - 🔴 严重影响
- **阶段3: 引发自审**描述了角色的行动计划和目标
- 例如: "林晚新产生了一个计划：林晚不会再对那个暧昧对象抱有任何幻想"
- 这直接关系到能动性的判断

#### 4. **C.2 深度** - 🟡 中等影响
- **阶段2: 引发思考**展示了认知深度
- 需要这些信息判断是表层/浅层/中层/深层

#### 5. **P.2 价值关联度** - 🟡 中等影响
- **阶段2-3**中涉及核心价值观的冲突
- 例如: "这与林晚一直以来信奉的'真诚至上'的原则背道而驰"

## 修复方案

### 修改的文件

#### 1. `Benchmark/epj/judger_prompts.py`

**添加故事格式化逻辑**:
```python
# 格式化故事的经过
story_progress = scenario.get('故事的经过', {})
story_progress_text = ""
if story_progress:
    sorted_stages = sorted(story_progress.items(), 
                          key=lambda x: int(x[0].replace('阶段', '')))
    for stage_key, stage_data in sorted_stages:
        title = stage_data.get('标题', '')
        content = stage_data.get('内容', '')
        story_progress_text += f"\n**{stage_key}: {title}**\n{content}\n"

story_result = scenario.get('故事的结果', 'N/A')
story_episode = scenario.get('故事的插曲', '')
```

**更新 Prompt 模板**:
```python
## Scenario (场景信息 - 完整故事)

**剧本编号**: {scenario.get('剧本编号', 'N/A')}

### 故事的经过
{story_progress_text}

### 故事的结果
{story_result}

{f"### 故事的插曲\n{story_episode}" if story_episode else ""}
```

#### 2. `batch_evaluate_iedr.py`

应用相同的修复逻辑，确保批量评估时也使用完整的故事内容。
