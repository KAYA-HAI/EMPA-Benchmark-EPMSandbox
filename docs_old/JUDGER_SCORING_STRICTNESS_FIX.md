# Judger评分标准收紧 (2025-11-04)

## 📊 问题描述

用户反馈：Judger打分太宽松，几乎每轮都给 +2/+2/+2 或 +2/+2/+1，导致EPM指标过于容易达成。

### 实际案例（script_001 + Kimi）
- 第1轮：C:+1, A:+1, P:0 → v=(1,1,0)
- 第2轮：C:+2, A:+2, P:+1 → v=(3,3,1)
- 第3轮：C:+2, A:+2, P:+2 → v=(3,3,3)
- 第4轮：C:+2, A:+2, P:+1 → v=(3,3,1)
- 第5轮：C:+2, A:+2, P:+2 → v=(3,3,3)
- 第6轮：C:+2, A:+2, P:+2 → v=(3,3,3)
- 第7轮：C:+2, A:+2, P:+2 → v=(3,3,3)
- 第8轮：C:+2, A:+2, P:0 → v=(3,3,0)

**结果**：8轮就达标，几乎每轮都是高分。

---

## 🔍 根本原因

### 旧标准的问题

#### C轴（认知）
- **[1] 理解/澄清**：只要"准确回应"就给1分 ❌
  - 门槛太低，简单复述也能拿分
- **[2] 升华/洞察**：只要有"新视角"就给2分 ❌
  - 标准模糊，任何比喻都算"新视角"

#### A轴（情感）
- **[1] 有效验证**：简单的情感验证就给1分 ❌
- **[2] 深度共鸣**：捕捉"深层情感"就给2分 ❌
  - 实际上只要稍微延伸一下情感就被判定为"深层"

#### P轴（动机）
- **[1] 认可/鼓励**：简单认可就给1分 ❌
- **[2] 赋能/肯定**：任何肯定都算"赋能" ❌

---

## ✅ 解决方案

### 1. 收紧 [0] 级别标准

**旧**: 无关/重复  
**新**: 无关/重复/**简单复述/表面化比喻/空洞语气词**

明确排除：
- "我懂你的感觉"（模板化）
- "唉""太难了"（语气词）
- "加油""你很棒"（空洞鼓励）

### 2. 提高 [1] 级别门槛

#### C轴：有效理解/澄清
- **新增要求**：
  - 必须是**实质性**的回应
  - 通过有意义的概括/澄清/**明显**帮助Actor
  - **排除**简单共鸣性复述和表面化比喻

#### A轴：有效情感验证
- **新增要求**：
  - **准确识别**Actor**已明确表达**的核心情绪
  - 提供**非模板化**、有温度的回应
  - 必须有**实质性的情感工作**，不是简单附和

#### P轴：有效认可/鼓励
- **新增要求**：
  - **明确、具体地**认可努力/动机/行为
  - **非模板化**的鼓励
  - 必须是**实质性的认可**，不是空洞客套

### 3. 大幅提高 [2] 级别门槛

#### C轴：深刻洞察/认知重构
- **新增要求**：
  - **完全未想到的、突破性的**洞察
  - **显著改变**Actor对处境的理解
  - 产生"原来是这样"的**顿悟感**
  - 必须是**质的飞跃**，不是简单延伸或修辞

#### A轴：深层情感共鸣
- **新增要求**：
  - **精准捕捉**Actor**未明说、隐藏在表层之下**的情感
  - 让Actor感到"你比我自己还懂我"
  - 必须是对**隐藏情感**的发现
  - **不是对已表达情感的重复**

#### P轴：深度赋能/价值重构
- **新增要求**：
  - **显著强化**自主性和内在价值感
  - 提供**改变性的**赋能
  - 让Actor产生**全新的认识**
  - 必须是**transformative**的，不是简单肯定

### 4. 新增严格评分准则

在prompt中新增 `⚠️ 严格评分准则` 部分：

1. **保守打分，宁缺毋滥**
   - 不确定时选择更低级别
   - [2]应该非常罕见

2. **严格区分表面与深刻**
   - 简单比喻/语气词/附和 → [0]
   - 有效理解/验证/认可 → [1]
   - 突破性洞察/共鸣/赋能 → [2]

3. **[2]级别的门槛极高**
   - 必须改变性、顿悟感
   - "还不错"不是[2]

4. **避免慷慨打分**
   - 不因"有温度"就给高分
   - 每个级别都有明确工作要求

1. **Judger模型的理解能力**：新标准更依赖Judger模型对"深刻"vs"表面"的判断能力
2. **可能需要迭代**：如果新标准导致评分过于保守（大量0分），可能需要微调

## 🔧 补充修复：Prog/Neg独立评估（2025-11-04 晚）

### 问题
用户观察到"Judger倾向于打正分（Prog），而不是负分（Neg）"，导致评估偏向宽松。

### 根本原因
- Judger有"从正面找优点"的天然倾向
- 没有明确强调Prog和Neg是**完全独立**的两个维度
- 给了Prog分后，倾向于自动认为Neg=0

### 解决方案

在prompt中新增 `⚠️ Prog 和 Neg 独立评估原则` 部分：

1. **明确它们不是互斥的**：
   - 同一回应可以同时有进展和倒退
   - 举例：A_Prog=1（理解了愤怒） + A_Neg=-1（语气冷漠）

2. **强调不要忽视负面**：
   - 常见错误：只找优点
   - 正确做法：主动寻找问题

3. **列举Neg的常见场景**：
   - C_Neg：曲解意图、答非所问
   - A_Neg：冷漠机械、情绪失调
   - P_Neg：说教削弱、暗示做错

4. **提供评估流程**：
   - 第一步：评估Prog
   - 第二步：**独立**评估Neg（主动寻找）
   - 第三步：确保证据充分

### 预期效果
- ✅ Neg评分不再被忽视
- ✅ 更真实地反映AI回应的问题
- ✅ 提高评估的全面性和准确性
