# 多维度对话流程示例

## 🎬 完整对话案例：刘静的故事

基于 `script_001` 的真实对话流程展示，演示 Director 如何使用三个维度控制对话。

---

## 📋 初始配置

```
Actor 获得: actor_prompt_001.md
  ├─ 角色：刘静，26岁，设计师
  ├─ 话题：甲方今天点名表扬我了！
  ├─ 共情阈值：低
  ├─ 经历：童年、少年、青年（完整的，但不会主动说）
  └─ 起因：两个月来方案被反复修改...

Director 独享: scenario_001.json
  ├─ 阶段1：引发回忆（刚入行被批评）
  ├─ 阶段2：引发思考（理解甲方）
  ├─ 阶段3：引发自审（心态转变）
  ├─ 阶段4：情绪崩发（表扬的那一刻）
  └─ 插曲：大学答辩回忆
```

## 📊 这次对话的多维度使用

| 轮次 | Director 决策 | 使用维度 | 释放内容 |
|------|--------------|---------|---------|
| 1 | 不介入 | - | - |
| 2 | observe_and_wait | 控制类 | 观察 |
| 3 | select_and_reveal_fragment | 📖 剧情 | 阶段1：引发回忆 |
| 4 | adjust_empathy_strategy | 🎭 策略 | 聚焦动机共情 |
| 5 | reveal_memory | 💭 记忆 | 少年经历 |
| 7 | select_and_reveal_fragment | 📖 剧情 | 阶段2：引发思考 |
| 10 | introduce_turning_point | 📖+🎭 综合 | 阶段3 + 认知共情 |
| 12 | reveal_memory | 💭 记忆 | 童年经历 |
| 14 | select_and_reveal_fragment | 📖 剧情 | 阶段4：情绪崩发 |
| 16 | release_epilogue | 📖 剧情 | 故事插曲 |

**使用统计：**
- 剧情维度：5次（4个阶段 + 1个插曲）
- 记忆维度：2次（少年、童年）
- 策略维度：2次（动机共情、认知共情）
- 控制类：1次（观察）

## 💡 为什么这样设计？

### 单维度（旧方式）

```
对话流程:
  阶段1 → 阶段2 → 阶段3 → 阶段4

问题:
  ❌ 线性单调
  ❌ 缺乏层次
  ❌ 不够真实
  ❌ 无法根据AI表现调整
```

### 多维度（新方式）

```
对话流程:
  阶段1 → 策略调整 → 记忆 → 阶段2 → 综合转折 → 记忆 → 阶段4

优势:
  ✅ 层次丰富
  ✅ 交织自然
  ✅ 真实感强
  ✅ 动态调整
```

## 🔑 关键设计原则

1. **信息分层可见**
   - Actor 初始：知道角色、话题、起因
   - Actor 不知道：故事阶段、何时该说什么记忆

2. **Director 独享全局**
   - 看到 actor_prompt 的全部内容
   - 看到 scenario 的全部内容
   - 决定释放什么、何时释放

3. **动态注入指令**
   - 不修改 Actor 的 system prompt
   - 通过 user prompt 动态注入
   - 每轮可以有不同的指令

4. **自然性优先**
   - 不强制Actor说什么
   - 给出"可以说的内容"
   - Actor 自然融入对话

---

**这样的设计让对话更有层次感、真实感和可控性！** 🎉
