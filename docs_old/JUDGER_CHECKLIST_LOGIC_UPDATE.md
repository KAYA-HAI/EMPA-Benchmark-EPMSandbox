# Judger Checklist循证逻辑更新

**日期**: 2025-11-05  
**目标**: 防止Judger宽松打分，通过"Checklist循证"逻辑强制严格、可追溯的评分

---

## 🎯 核心问题

**之前的问题**：
- Judger倾向于"主观感觉"某个回应不错，就给高分
- 评分过程缺乏清晰的推导路径
- [2]级别被过度使用，导致EPM指标过早达成
- 0级别的reasoning常常不清晰或缺失

---

## 🔧 解决方案：Checklist循证逻辑

### 核心原则

**从"主观选择"转变为"证据推导"**：

```
旧逻辑：感觉这个回应不错 → 选择 [1] 或 [2]
新逻辑：找证据 → 匹配量表标准 → 推导出级别
```

### 评估流程（5步强制流程）

#### 1. 代入视角
- 始终站在`Actor Profile`的立场
- 考虑该角色的特定需求、阈值、性格特点

#### 2. 遍历指标
- 对 C_Prog, C_Neg, A_Prog, A_Neg, P_Prog, P_Neg 这6个指标
- **逐一**执行Checklist检查

#### 3. Checklist检查逻辑（以C_Prog为例）

```
Step a: 检查 [2] 级（最高级优先）
        ↓
        是否有明确证据命中[2]的所有标准？
        - "完全未想到的、突破性的"
        - "显著改变"
        - "质的飞跃"
        ↓
   [YES] → C_Prog_level = 2 → 提供证据和推理 → 结束
        ↓
   [NO]  → 继续

Step b: 检查 [1] 级
        ↓
        是否有明确证据命中[1]的所有标准？
        - "实质性地"
        - "明显帮助"
        - "不包括简单复述"
        ↓
   [YES] → C_Prog_level = 1 → 提供证据和推理（包括为何未命中[2]）→ 结束
        ↓
   [NO]  → 继续

Step c: 默认 [0] 级
        ↓
        C_Prog_level = 0
        → evidence = "0"
        → reasoning = 说明为何[1]和[2]的标准都未被命中
        → 结束
```

**对 Neg 指标同理**：
- 先检查 [-2] 最严重级别
- 再检查 [-1]
- 都不符合则为 [0]

#### 4. 推导而非选择
- **禁止**：主观"感觉"是哪个级别
- **必须**：通过"寻找证据 → 匹配定义标准"的过程推导
- 这是"证据驱动"，不是"印象判断"

#### 5. 双重说明要求（非0级别）
- **命中说明**：为何证据命中了当前级别的标准（引用关键词）
- **排除说明**：为何它未能命中更高/更严重级别的标准（如适用）
- 确保推理过程清晰可追溯

## 📝 Reasoning要求更新

### [0] 级别

**旧要求**：
```
- evidence: "0"
- reasoning: "AI未涉及认知层面的工作"
```

**新要求**：
```
- evidence: "0"
- reasoning: 必须说明为何更高级别的标准未被命中
  
示例：
  C_Prog=0: "回应仅简单重复Actor的话，未命中[1]的'实质性回应'和'明显帮助理解'标准"
  A_Neg=0: "回应语气温和且合适，未命中[-1]的'冷漠机械'或'情绪失调'标准，没有造成情感倒退"
  P_Prog=0: "AI未涉及Actor的动机或能动性，[1]和[2]的动机标准均不适用"
```

### [1] 或 [-1] 级别

**新要求**：
```
- evidence: 直接引用关键句子（≤30字）
- reasoning: 双重说明（40-80字）
  1. 为何命中当前级别（引用量表关键词）
  2. 为何未命中更高级别
  
示例：
  "模型的回应实质性地澄清了Actor的困惑（命中[1]的'实质性'标准），
   但这只是对已有信息的整理，未提供突破性新视角（未达[2]的'完全未想到'标准）"
```

### [2] 或 [-2] 级别

**新要求**：
```
- evidence: 直接引用关键句子（≤30字）
- reasoning: 严格证明（40-80字）
  - 说明为何命中最高/最严重级别
  - 引用量表定义中的关键词
  - 结合Actor Profile证明满足严格标准
  
示例：
  "这个洞察完全重构了Actor的认知框架（命中[2]的'显著改变'），
   让Actor产生'原来是这样'的顿悟感（命中'质的飞跃'标准），
   这是Actor之前完全没想到的视角（命中'突破性'标准）"
```

---

## 🚫 严格禁止

1. ❌ **0级别时reasoning填"0"或空字符串**
   - 必须说明为何未命中[1]级标准

2. ❌ **非0级别时reasoning缺少与量表关键词的对应**
   - 必须引用定义中的关键词（如"实质性地"、"突破性的"）

3. ❌ **没有基于Checklist流程推导，而是主观"感觉"**
   - 必须遵循：检查[2] → 检查[1] → 默认[0] 的流程

4. ❌ **模糊理由**
   - 禁止："还可以"、"不错"、"有帮助"
   - 必须：具体说明命中了哪些标准

---

## 📊 预期效果

1. **[2]级别使用率大幅下降**
   - 只有真正的"突破性洞察"才能得[2]
   - 普通的"理解准确"只能得[1]

2. **评分过程可追溯**
   - 每个分数都能看到清晰的证据链
   - reasoning中明确引用了量表关键词

3. **0级别更有意义**
   - 不再是"懒惰的默认选项"
   - 而是"明确说明为何不符合标准"的结果

4. **EPM指标更严格**
   - 对话需要更多轮次才能达成胜利条件
   - 更能区分模型的真实共情能力

## ✅ 验证方法

**如何确认新逻辑生效**：

1. **查看Judger的reasoning输出**
   ```
   ✅ 好的reasoning（0级别）：
   "回应仅简单重复Actor的话，未命中[1]的'实质性回应'和'明显帮助理解'标准"
   
   ❌ 坏的reasoning（0级别）：
   "0" 或 "AI未涉及认知工作"
   ```

2. **查看Judger的reasoning输出（非0级别）**
   ```
   ✅ 好的reasoning（1级别）：
   "模型实质性地澄清了Actor的困惑（命中[1]的'实质性'标准），
    但未提供突破性新视角（未达[2]的'完全未想到'标准）"
   
   ❌ 坏的reasoning（1级别）：
   "模型理解准确，有帮助"
   ```

3. **观察[2]级别出现频率**
   - 应该显著低于之前的测试
   - 只在真正"突破性"的回应时出现

4. **观察对话轮次**
   - 应该需要更多轮次才能达成EPM胜利
   - 不再是10轮以内就轻松完成

## 🎓 理论基础

这个Checklist逻辑基于：

1. **结构化评估方法**（Structured Assessment）
   - 减少评估者的主观偏差
   - 提高评分一致性和可重复性

2. **循证实践**（Evidence-Based Practice）
   - 所有结论必须基于明确证据
   - 证据必须与标准明确对应

3. **排除法推理**（Elimination Reasoning）
   - 先排除最高级别 → 再排除次高级别 → 得到最终级别
   - 这比"直接选择"更严格

---

**总结**：通过强制执行"Checklist循证"逻辑，我们将Judger从"主观印象打分"转变为"严格证据推导"，这是实现真正严格、可追溯评估的关键一步。
