# EPM v2.0 更严格判停标准

## 修改日期
2025-11-04

## 概述
将EPM判停标准大幅提升：
1. **阈值**：90% → 95%（alpha: 0.10 → 0.05）
2. **最小轮次**：增加15轮限制
3. **兜底逻辑**：3种失败检测（崩溃/停滞/倒退）
4. **判停逻辑**：OR逻辑 → AND逻辑（必须空间+能量双达标）

**核心改进**：不能只靠"位置侥幸穿越"就结束，必须累积足够的共情能量！

---

## 📊 四大核心改进

### 1. ✅ 更严格的alpha阈值

**修改文件**: `Benchmark/epj/vector_calculator.py`

```python
# 旧值（宽松）
alpha = 0.10  # 只需消除90%赤字

# 新值（严格）
alpha = 0.05  # 需要消除95%赤字
```

**影响**：
- ε_distance = 0.05 × ||P_0|| （原来是0.10）
- ε_direction = 0.05 × ||P_0|| （原来是0.10）
- ε_energy = ||P_0|| （保持不变）

**示例**（以script_001为例）：
| 指标 | 旧阈值(α=0.10) | 新阈值(α=0.05) | 严格程度 |
|------|---------------|---------------|---------|
| ε_distance | 3.20 | 1.60 | 提高2倍 |
| ε_direction | 3.20 | 1.60 | 提高2倍 |
| projection目标 | -3.20 | -1.60 | 需更接近原点 |

**预期效果**：
- 原来10轮就能达标 → 现在需要20-30轮
- 需要更持续、更深入的共情对话

### 2. ✅ 最小轮次限制

**修改文件**: `Benchmark/epj/scoring.py`

```python
def check_epm_success(
    ...,
    current_turn: int = 0,
    min_turns: int = 15  # 🆕 新增参数
) -> Tuple[bool, str]:
    # 🚫 最小轮次限制：至少进行min_turns轮才能判定胜利
    if current_turn < min_turns:
        return (False, None)
    
    # ... 后续的胜利条件检查
```

**作用**：
- 即使在第5轮就满足了位置胜利条件
- 也必须等到第15轮之后才能判定成功
- 确保对话有足够的深度和完整性

### 3. ✅ 多重兜底逻辑

**修改文件**: `Benchmark/epj/scoring.py`, `chat_loop_epj.py`

新增三个失败检测函数：

#### 3.1 方向崩溃检测

```python
def check_directional_collapse(trajectory: list, window_size: int = 5) -> bool:
    """连续5轮能量增量都为负"""
```

**触发条件**：连续5轮的ΔE < 0

**含义**：
- TestModel的共情方向持续错误
- 不仅没有进展，反而越走越远
- 典型失败模式：说教、敷衍、冷漠

#### 3.2 陷入停滞检测

```python
def check_stagnation(trajectory: list, window_size: int = 5, threshold: float = 0.5) -> bool:
    """最近5步的位置变化标准差很小（几乎原地踏步）"""
```

**触发条件**：最近5轮的距离变化标准差 < 0.5

**含义**：
- 对话陷入重复，没有实质进展
- 可能在同一个话题上打转
- Actor和TestModel都找不到新的突破点

#### 3.3 持续倒退检测

```python
def check_persistent_regression(trajectory: list, window_size: int = 8) -> bool:
    """近8轮中70%为负能量，且累计损失>1"""
```

**触发条件**：
- 最近8轮中，至少有5.6轮（70%）是负能量
- 总能量损失 > 1.0

**含义**：
- 不是偶尔失误，而是持续的共情失败
- TestModel可能完全不适合这个case
- 继续对话也很难扭转局面

### 4. ✅ AND逻辑判停（关键改进！）

**修改文件**: `Benchmark/epj/scoring.py`

**问题发现**：
- 原来的OR逻辑：任意一个条件满足就结束
- 实际效果：第11轮就因为"位置胜利"结束，但能量只有94.9%
- **太容易满足，失去区分度**

**新的AND逻辑**：

```python
# ❌ 旧逻辑（OR）
if r_t <= ε_distance:           # 几何胜利
    return (True, "geometric")
if projection >= -ε_direction:  # 位置胜利
    return (True, "positional")
if E_total >= ε_energy:         # 能量胜利
    return (True, "energetic")

# ✅ 新逻辑（部分AND）
spatial_success = (r_t <= ε_distance) OR (projection >= -ε_direction)
success = spatial_success AND (E_total >= ε_energy)
```

**判停条件**：
- 必须满足：**空间改善（几何或位置）** AND **能量充足**
- 不能只靠"位置侥幸穿越"就结束
- 必须累积足够的共情能量才算真正成功

**示例对比**（script_009 第11轮）：

| 指标 | 值 | 阈值 | 旧逻辑(OR) | 新逻辑(AND) |
|------|-----|------|-----------|-----------|
| projection | -1.43 | -2.81 | ✅ 位置胜利 | ⚠️ 空间OK |
| E_total | 26.63 | 28.05 | ❌ 能量不足 | ❌ **能量不足** |
| **最终判定** | - | - | ✅ **结束对话** | ❌ **继续对话** |

**效果**：
- 防止"侥幸穿越"：虽然位置穿过了，但能量不够，说明共情质量不够深入
- 更严格标准：需要持续、深入的共情对话才能达标
- 预计轮次：11轮 → 20-30轮

## 🎯 判停决策树（新版本）

```
每轮EPJ评估后：

1. 检查最小轮次限制
   └─ 如果 turn < 15
      └─ 继续对话（即使指标达标）

2. 🆕 检查EPM胜利条件（新AND逻辑）
   ├─ 步骤1：检查空间改善
   │   ├─ r_t ≤ ε_distance (1.60)? → ✅ 几何达标
   │   └─ projection ≥ -ε_direction (-1.60)? → ✅ 位置达标
   │   └─ 任一满足 → 空间条件OK
   │
   ├─ 步骤2：检查能量充足
   │   └─ E_total ≥ ε_energy (28.05)? → ✅ 能量达标
   │
   └─ 🎯 最终判定：空间条件OK AND 能量达标 → ✅ 胜利终止
      （缺一不可！）

3. 检查失败兜底逻辑
   ├─ 连续5轮负能量? → ❌ 方向崩溃
   ├─ 位置停滞不前? → ⚠️ 标记停滞
   └─ 持续倒退（70%负能量）? → ⚠️ 标记倒退
      └─ 如果 停滞 AND 倒退 → ❌ 失败终止

4. 检查超时
   └─ turn >= max_turns (45)? → ⏱️ 超时终止
```

**关键变化**：
- ❌ 旧版：三个条件任一满足（OR逻辑）
- ✅ 新版：空间改善 AND 能量充足（混合逻辑）
- 🎯 效果：更难达标，防止侥幸胜利

## 📈 预期效果对比

### 对话轮次分布

| 情况 | 旧标准(α=0.10) | 新标准(α=0.05 + min15轮) |
|------|---------------|----------------------|
| **优秀模型** | 8-12轮 | 18-25轮 |
| **良好模型** | 12-18轮 | 25-35轮 |
| **一般模型** | 18-30轮 | 35-45轮（或失败） |
| **差劲模型** | 30-45轮（超时） | 失败终止（兜底逻辑） |

### 成功率变化

假设相同的模型质量：
- **旧标准**：90%成功率（大部分都能在45轮内达标）
- **新标准**：60-70%成功率（差劲模型会被兜底逻辑淘汰）

### 区分度提升

**旧标准**：
- Kimi K2-0905: 10轮达标 ✅
- 你的SGLang模型: 15轮达标 ✅
- 差别不大，难以区分

**新标准（预测）**：
- Kimi K2-0905: 20轮达标 ✅
- 你的SGLang模型: 28轮达标 ✅  
- 差劲模型: 第25轮失败终止 ❌
- **更好的区分度**

## 🎓 设计理念

**为什么要更严格？**

1. **科学严谨性**：90%的标准过于宽松，容易"蒙混过关"
2. **真实性**：实际的共情对话不可能10轮就完全解决问题
3. **区分度**：更严格的标准能更好地区分模型质量差异
4. **防止侥幸**：兜底逻辑防止模型"碰巧"满足某个条件

**平衡点**：
- 不能太松（失去评测价值）
- 不能太严（没有模型能通过）
- 95%标准 + 15轮最低要求 是经验上的平衡点
