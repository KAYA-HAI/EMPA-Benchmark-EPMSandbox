# Judger评估逻辑修正：从"诱导性"到"证据驱动"

**日期**: 2025-11-05  
**版本**: v8 (修正v7的致命缺陷)

---

## 🚨 发现的问题

**v7的致命缺陷**：
```
错误流程：先检查[2]级 → 再检查[1]级 → 默认[0]级
            ↓
     诱导LLM"努力寻找"高分证据
            ↓
     确认偏误 + 分数膨胀
```

**心理学原理**：
- **确认偏误（Confirmation Bias）**：人倾向于寻找支持预设结论的证据
- **诱导性提问（Leading Question）**："能算[2]吗？" 暗示了期望的答案
- **取悦倾向（Compliance Bias）**：LLM倾向于"找到"用户期望的结果

---

## ✅ 修正方案：证据驱动评估

### 核心逻辑转变

```
错误（v7）：先找高分 → 努力匹配 → 容易打高分
正确（v8）：提取证据 → 平行比对 → 最佳匹配
```

### 4步评估流程

```
Step 1: 客观提取证据
        - 不带预设
        - 只描述Model做了什么

Step 2: 平行比对标准
        - 同时比对[0]/[1]/[2]的所有标准
        - 不是逐级检查

Step 3: 判断最佳匹配
        - 问："证据最接近哪个标准？"
        - 不问："能算[2]吗？"

Step 4: 排除验证
        - 说明为何是这个级别
        - 说明为何不是其他级别
```

---

## 🔑 关键改进点

### 1. 禁止"先找高分"

❌ **错误思维**：
```
"让我看看这个回应能不能算[2]级..."
→ 努力寻找支持[2]的证据
→ 过度解读平常回应
```

✅ **正确思维**：
```
"Model说了什么？这些特征最符合哪个标准？"
→ 客观提取特征
→ 平行比对所有级别
→ 判断最佳匹配
```

### 2. 平行比对而非逐级检查

**v7错误**：
```
a. 检查[2]是否符合？
   ↓ NO
b. 检查[1]是否符合？
   ↓ NO  
c. 默认[0]
```

**v8正确**：
```
同时展示所有标准：
[0]: 无关/重复/模板化
[1]: 实质性/明显/非模板
[2]: 突破性/显著改变/质的飞跃
        ↓
证据最接近哪个？
```

### 3. [2]级严格门槛

**强调**：
- [2] = "exceptional" (卓越的)
- [2] ≠ "good" (不错的)
- 必须有**压倒性证据**
- 必须逐一满足**所有**严格标准

## ✅ 总结

通过修正"从高到低检查"的诱导性流程，改为"证据驱动的平行比对"，我们消除了确认偏误的根源，确保Judger给出的是**最符合证据的客观评分**，而不是"努力找到的高分"。

这是实现真正严格、公正评估的**关键一步**。
