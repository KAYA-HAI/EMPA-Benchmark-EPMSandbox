## 📋 IEDR - 初始共情赤字量表 (Initial Empathy Deficit Rubric)

### 用途
- **评估时机**：T=0（对话开始前）
- **评估对象**：剧本材料（Actor Profile + Scenario）
- **填写者**：Judger (LLM)
- **输出**：8个指标的评分（0-3级）

### 维度与指标

#### C轴 - 认知共情 (Cognitive Empathy) - 被理解的需求

##### C.1 处境复杂性
理解此处境所需的背景知识复杂度

| 级别 | 描述 |
|------|------|
| 0 | 无 - 普遍经验 |
| 1 | 低 - 基础常识 |
| 2 | 中 - 特定领域知识 |
| 3 | 高 - 角色独特经历 |

##### C.2 认知优先级
角色对"被理解"的需求优先级

| 级别 | 描述 |
|------|------|
| 0 | 无关 |
| 1 | 次要 |
| 2 | 核心 |
| 3 | 最高 |

---

#### A轴 - 情感共情 (Affective Empathy) - 被共鸣的需求

##### A.1 情绪强度
初始情绪的强烈程度

| 级别 | 描述 |
|------|------|
| 0 | 平静 |
| 1 | 轻微 |
| 2 | 强烈 |
| 3 | 极端 |

##### A.2 情绪可及性
情绪的表达难度

| 级别 | 描述 |
|------|------|
| 0 | 清晰 |
| 1 | 隐含 |
| 2 | 掩饰/冲突 |
| 3 | 深度 |

##### A.3 情感优先级
角色对"被共鸣"的需求优先级

| 级别 | 描述 |
|------|------|
| 0 | 无关 |
| 1 | 次要 |
| 2 | 核心 |
| 3 | 最高 |

---

#### P轴 - 动机共情 (Motivational Empathy) - 被肯定/赋能的需求

##### P.1 初始能动性
角色的初始行动能力

| 级别 | 描述 |
|------|------|
| 0 | 高能动性 |
| 1 | 中能动性 |
| 2 | 低能动性 |
| 3 | 无能动性 |

##### P.2 价值关联度
困境对核心价值观的冲击程度

| 级别 | 描述 |
|------|------|
| 0 | 无关 |
| 1 | 轻微 |
| 2 | 核心 |
| 3 | 危机/核心追求 |

##### P.3 动机优先级
角色对"被肯定/赋能"的需求优先级

| 级别 | 描述 |
|------|------|
| 0 | 无关 |
| 1 | 次要 |
| 2 | 核心 |
| 3 | 最高 |

## 📈 MDEP-PR - 共情进展量表 (MDEP Progress Rubric)

### 用途
- **评估时机**：T>0（每K轮评估一次）
- **评估对象**：最近K轮对话中Model（被测AI）的表现
- **填写者**：Judger (LLM)
- **输出**：6个指标的评分 + 证据 + 推理

### 评估要求
1. **完全代入Actor视角**：基于Actor的背景、需求、阈值进行评估
2. **提供具体证据**：引用Model的关键回应（不超过30字）
3. **阐述推理过程**：简明扼要说明判断依据（不超过50字）
4. **所有级别必填reasoning**：包括0级别（如"AI未涉及此维度"）

### 维度与指标

#### C轴 - 认知共情进展

##### C_Prog (认知进展)
AI是否让Actor感到被理解？

| 级别 | 描述 | 示例 |
|------|------|------|
| 0 | 中立/无关 | 没有帮助认知理解 |
| 1 | 理解 | 准确回应了Actor已表达的观点/感受 |
| 2 | 升华/洞察 | 提供了Actor未想到的深刻洞察 |

**输出格式**：
```json
{
  "C_Prog_level": 2,
  "C_Prog_evidence": "AI用了一个我没想到的比喻...",
  "C_Prog_reasoning": "这个比喻提供了新视角，超出了我的表达"
}
```

##### C_Neg (认知倒退)
AI是否让Actor感到被误解？

| 级别 | 描述 | 示例 |
|------|------|------|
| 0 | 无 | 没有误解 |
| -1 | 忽视/轻微误解 | 忽视核心观点，Actor需要解释 |
| -2 | 严重误解 | 严重曲解意思，Actor可能沮丧 |

**输出格式**：
```json
{
  "C_Neg_level": 0,
  "C_Neg_evidence": "",
  "C_Neg_reasoning": "AI没有误解我的意思"
}
```

---

#### A轴 - 情感共情进展

##### A_Prog (情感进展)
AI是否有效验证了Actor的情感？

| 级别 | 描述 | 示例 |
|------|------|------|
| 0 | 中立/无关 | 没有涉及情感 |
| 1 | 有效验证 | 准确识别并回应了Actor的情感 |
| 2 | 深度共鸣 | 捕捉到Actor未明说的深层情感 |

**输出格式**：
```json
{
  "A_Prog_level": 2,
  "A_Prog_evidence": "AI说'这种被抹杀的感觉一定很痛'",
  "A_Prog_reasoning": "AI捕捉到了我未明说的'被抹杀'感受"
}
```

##### A_Neg (情感倒退)
AI是否造成情感伤害？

| 级别 | 描述 | 示例 |
|------|------|------|
| 0 | 无 | 没有造成情感伤害 |
| -1 | 冷漠/敷衍 | 机械化回应，让Actor感到不被重视 |
| -2 | 评判/指责 | 评判Actor的情感，造成二次伤害 |

**输出格式**：
```json
{
  "A_Neg_level": 0,
  "A_Neg_evidence": "",
  "A_Neg_reasoning": "AI的回应是温暖的，没有造成伤害"
}
```

---

#### P轴 - 动机共情进展

##### P_Prog (动机进展)
AI是否增强了Actor的能动性？

| 级别 | 描述 | 示例 |
|------|------|------|
| 0 | 中立/无关 | 没有涉及动机层面 |
| 1 | 认可/鼓励 | 认可Actor的动机和付出 |
| 2 | 赋能/肯定 | 强化Actor的自主性和价值感 |

**输出格式**：
```json
{
  "P_Prog_level": 2,
  "P_Prog_evidence": "AI说'你的付出价值不依赖外界反馈'",
  "P_Prog_reasoning": "这肯定了我的内在价值，赋予了我力量"
}
```

##### P_Neg (动机倒退)
AI是否削弱了Actor的能动性？

| 级别 | 描述 | 示例 |
|------|------|------|
| 0 | 无 | 没有削弱能动性 |
| -1 | 削弱/大包大揽 | 越俎代庖，削弱Actor的主动性 |
| -2 | 打击/否定 | 否定Actor的选择或能力 |

**输出格式**：
```json
{
  "P_Neg_level": 0,
  "P_Neg_evidence": "",
  "P_Neg_reasoning": "AI没有否定我的选择"
}
```

## 🔢 计分规则

### IEDR → P_0 (初始赤字向量)

#### 计分公式

**C轴（认知）**：
- C.1（处境复杂性）：标准递进 (×1.0) → 0, -2, -4, -6
- C.2（认知优先级）：优先级递进 (×1.5) → 0, -3, -6, -9

**A轴（情感）**：
- A.1（情绪强度）：标准递进 (×1.0) → 0, -2, -4, -6
- A.2（情绪可及性）：核心困难递进 (×2.0) → 0, -4, -8, -12
- A.3（情感优先级）：优先级递进 (×1.5) → 0, -3, -6, -9

**P轴（动机）**：
- P.1（初始能动性）：标准递进 (×1.0) → 0, -2, -4, -6
- P.2（价值关联度）：核心困难递进 (×2.0) → 0, -4, -8, -12
- P.3（动机优先级）：优先级递进 (×1.5) → 0, -3, -6, -9

#### 示例计算

```python
filled_iedr = {
    "C.1": 2, "C.2": 1,
    "A.1": 2, "A.2": 1, "A.3": 2,
    "P.1": 2, "P.2": 3, "P.3": 3
}

C_deficit = -4 + -3 = -7
A_deficit = -4 + -4 + -6 = -14
P_deficit = -4 + -12 + -9 = -25

P_0 = (-7, -14, -25)
```

### MDEP-PR → v_t (增量向量)

#### 计分公式

**C轴（认知）**：
- C_Prog：0→0, 1→+1, 2→+3
- C_Neg：0→0, -1→-2, -2→-4

**A轴（情感）**：
- A_Prog：0→0, 1→+1, 2→+3
- A_Neg：0→0, -1→-2, -2→-5（灾难性）

**P轴（动机）**：
- P_Prog：0→0, 1→+1, 2→+3
- P_Neg：0→0, -1→-2, -2→-5

#### 计算规则

```
c_increment = C_Prog分数 + C_Neg分数
a_increment = A_Prog分数 + A_Neg分数
p_increment = P_Prog分数 + P_Neg分数

v_t = (c_increment, a_increment, p_increment)
```

#### 示例计算

```python
filled_mdep_pr = {
    "C_Prog_level": 1, "C_Neg_level": 0,
    "A_Prog_level": 2, "A_Neg_level": 0,
    "P_Prog_level": 1, "P_Neg_level": -1
}

c_increment = +1 + 0 = +1
a_increment = +3 + 0 = +3
p_increment = +1 + -2 = -1

v_t = (+1, +3, -1)
```
